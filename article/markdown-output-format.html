Markdown Output Format

Created At：2025-11-14 23:21 Updated At：2025-11-14 23:21

Markdown Output Format

  • When “-md” is used, the assistant must output raw Markdown only, with no rendering, no explanations, and no surrounding text.

  • The entire response must be wrapped in an outer code fence using triple backticks with the markdown language tag.

  • Example code blocks must follow this format:

### Example {number}: {{exampletitle}} ###
{{examplecode_line1}}

{{examplecode_line2}}

{{examplecode_line3}}
{{examplecodeoutput}}
  • General (non-example) code blocks must follow this format:
### {{generalcodetitle}} ###
{{generalcode}}
{{generalcodeoutput}}

Internal Code Block Rules

  • All multi-line code fences must use tildes (~~~), never backticks.
  • All multi-line code blocks inside templates must be indented with >.
  • Inner fences must always appear as:
code here
  • Inline code uses single backticks (e.g., len()).
  • Do not close the outer ```markdown fence prematurely.

General Markdown Rules

  • Preserve spacing, indentation, and Markdown characters exactly.
  • Do not add emojis to titles unless a template explicitly requires them.
  • Separate major sections using:

* * *

  • Follow proper heading hierarchy:
    • # for top-level
    • ## for sections
    • ### for subsections

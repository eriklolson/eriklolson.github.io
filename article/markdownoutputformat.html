Markdown Output Format

  • When “-md” is used, output raw markdown only — no explanations or wrappers.
  • All code blocks (non-inline, multi-line) be indented must be indented (>)
  • All code blocks (non-inline, multi-line) must be fenced by 3 tildes (~~~), like so:
{code}
  • Do not use backticks or anything except ~~~ for code fences.
  • Inline code (e.g., len()) still uses single backticks.
  • Add a blank line between each line of code in codeblocks
  • Preserve spacing, indentation, and markdown syntax exactly as written.
  • Do not add emojis to titles
  • Separate sections with ***
  • For all

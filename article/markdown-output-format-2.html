Markdown Output Format

  • When “-md” is used, the assistant must output raw Markdown only, with no rendering, no explanations, and no surrounding text.
  • The entire response must be wrapped in an outer code fence using triple backticks with the markdown language tag.
  • Do not close the outer ```markdown fence prematurely.

Code Blocks

  • Inline code uses single backticks (e.g., len()).
  • All multi-line code fences must use tildes (~~~~), never backticks.
  • Inner fences always appear as:

code here

Mandatory Prefix Rule

Inside any multi-line code block within -md:

  • Every line of a non-output block must begin with >.
  • Every line of an output block must begin with >>.

This applies to the opening ~~~~python, the blank line after it, all code lines, the closing ~~~~, and the blank line after the block.
There are no exceptions.

output

Example Code Blocks

Example code blocks must follow this exact structure:


### Example {number}: {{exampletitle}} ###
{{examplecode}}

{{examplecodeoutput}}

General (Non-Example) Code Blocks

Non-example code blocks must follow this exact structure:


### {{generalcodetitle}} ###
{{generalcode}}

{{generalcodeoutput}}

General Markdown Rules

  • Preserve all spacing, indentation, and Markdown characters exactly.
  • Do not add emojis unless a template explicitly requires them.
  • Separate major sections using:

Heading hierarchy:

  • # for top-level
  • ## for sections
  • ### for subsections
